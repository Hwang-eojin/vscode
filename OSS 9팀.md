### 1. 개요

Visual Studio Code (이하 VSCode)는 마이크로소프트에서 개발한, 다양한 프로그래밍 언어를 지원하는 무료 코드 편집기이다. 강력한 코드 편집 및 디버깅 기능, 내장된 Git 컨트롤, 문법 강조 표시, 코드 완성, 코드 리팩토링, 그리고 확장 프로그램 지원 등의 기능을 제공하며, 이를 통해 개발자는 보다 효율적으로 코드를 작성, 검토 및 실행할 수 있다. VSCode는 Windows, macOS, 그리고 Linux 운영 체제에서 사용할 수 있다.

### 2. 라이선스

Visual Studio Code (이하 VSCode)는 마이크로소프트가 지원하는 오픈 소스 소프트웨어로, MIT 라이선스에 따라 배포된다. 이 라이선스 하에서 사용자는 VSCode를 자유롭게 사용, 복제, 수정, 배포할 수 있으며, 소프트웨어의 모든 복제본 또는 주요 부분에 저작권 및 허가 표시를 포함해야 한다. VSCode의 소스 코드는 GitHub에서 확인 가능하며, 개인이나 기업 모두 비용 없이 사용할 수 있다. 또한, MIT 라이선스는 저작권자가 소프트웨어 사용으로 인해 발생할 수 있는 어떠한 책임도 지지 않음을 명시하고 있으며, 상업적 사용 시에는 이에 유의해야 한다. 

Github에서 배포되는 소스 코드는 MIT 라이선스하에 배포되는 오픈 소스지만, 실제로 공식 사이트에서 배포되는 릴리즈 바이너리는 마이크로소프트의 독자적 라이선스 (EULA의 일종)하에 배포되는 것으로 오픈 소스가 아니다. 따라서 대부분 사용자들은 마이크로소프트의 독자적 라이선스 하에 Visual Studio Code를 사용하고 있다. 이를 통해 사실상 이중 라이선스가 병존하고 있다고 볼 수 있다.

### 3-1. 주요기능 : VSCode의 확장 프로그램 (이성현)

##### VSCode 확장 프로그램이란?

- VSCode 확장 프로그램은 VSCode의 기본 기능을 확장하고 추가하는 소프트웨어 구성 요소
- Microsoft, VSCode 개발팀, 그리고 개발자 커뮤니티에서 다양한 확장 프로그램을 만들고 제공하고 있다.
- 이를 통해 VSCode 사용자들은 자신의 니즈에 맞게 편집기를 커스터마이징할 수 있다.

##### 확장 프로그램의 장점

- VSCode를 사용자 개개인의 요구사항과 선호도에 맞춰 최적화할 수 있다.
- 핵심 기능 외에도 광범위한 추가 기능과 통합을 제공받을 수 있다.
- 다양한 자동화 기능과 생산성 향상 도구를 활용할 수 있다.
- 제한적인 기본 편집기 기능에 얽매이지 않고 워크플로우를 개선할 수 있다.

##### 확장 프로그램이 유용한 경우

- 특정 프로그래밍 언어 또는 기술 스택을 사용할 때
- 외부 도구, 서비스, 워크플로우와의 통합이 필요할 때
- 반복 작업을 자동화하고 코딩 효율성을 높이고 싶을 때
- 팀원들과 협업하며 실시간 공동 작업이 필요할 때
- VSCode의 외관, 동작, 워크플로우를 개인 취향에 맞게 조정하고 싶을 때

### 3-2. 주요기능 : * (강지호)

##### 포맷팅과 정적 분석도구란?

코드 포맷팅은 코드의 형식을 일관되게 스타일을 자동으로 정렬하고 들여쓰기를 조정한다. VSCode는 내장된 포맷터와 확장을 통해 다양한 언어에 대한 자동 포맷팅을 지원한다.  개발자는 포맷팅 규칙을 사용자 정의할 수 있어서 자신이 선호하는 코드 스타일을 유지할 수 있다.

정적 분석은 코드를 실행하지 않고도 소스 코드를 분석하여 잠재적인 오류나 버그를 식별하는 과정이다. VSCode는 다양한 정적 분석 도구를 지원하여 코드 품질을 향상시키고 안정성을 높인다. 이러한 도구는 코드의 복잡성, 코드 중복, 잠재적인 버그, 보안 취약점 등을 식별하여 개발자에게 경고나 권장을 제공한다.

 ##### 장점

포맷팅 장점 : 코드를 작성하거나 편집하는 동안에는 포맷터가 코드를 자동으로 정렬하고 들여쓰기를 조정하여 가독성을 향상시킨다. 이를 통해 개발자는 일관된 코드 스타일을 유지할 수 있게 되고, 코드 리뷰 및 협업이 용이해진다.

정적 분석 장점 : 정적 분석을 통해 개발자는 오류를 미리 발견하고 수정하여 소프트웨어의 안정성과 신뢰성을 향상시킬 수 있다.

이러한 포맷팅과 정적 분석 도구는 개발자에게 코드 작성과 유지보수를 더욱 쉽고 효율적으로 만들어주는 강력한 도구이다. 코드의 품질과 안정성을 높이는 데 큰 도움이 된다.

##### Prettier 사용 및 기능

Prettier : 자동 포맷팅: 코드를 저장할 때마다 VSCode에서 Prettier가 자동으로 실행되어 코드를 일관된 스타일로 자동으로 포맷팅합니다. 이는 설정에서 editor.formatOnSave를 true로 설정함으로써 가능합니다.
수동 포맷팅: 필요한 경우 명령 팔레트(단축키: Ctrl+Shift+P)에서 "Format Document" 명령을 실행하여 코드를 수동으로 포맷할 수 있습니다.
선택적 사용: Prettier는 일반적으로 코드를 자동으로 포맷팅하지만, 설정에서 특정 파일에 대해 포맷터를 변경하거나 Prettier를 비활성화할 수도 있습니다.
Prettier를 사용하면 코드의 일관성을 유지하고 가독성을 높일 수 있습니다. 코드를 저장할 때마다 자동으로 실행되므로 코드를 포맷하는 과정이 자동화되어 개발자가 일일이 코드를 정리할 필요가 없어집니다.

##### ESLint에 주요 기능

ESLint: 
문제 식별: ESLint는 코드에서 발생하는 잠재적인 문제를 식별합니다. 이는 문법 오류, 잘못된 변수 사용, 잘못된 함수 호출 등을 포함합니다.
코드 스타일 검사: 코드를 일관된 스타일로 작성할 수 있도록 도와줍니다. 예를 들어, 들여쓰기 규칙, 따옴표 사용, 세미콜론 등의 스타일 가이드를 설정할 수 있습니다.
커스터마이징: ESLint는 다양한 플러그인과 규칙을 통해 사용자 정의할 수 있습니다. 이는 프로젝트의 요구 사항에 따라 코드 스타일을 맞출 수 있도록 합니다.
자동화: 대부분의 개발 환경에서 ESLint를 자동화하여 코드를 저장할 때마다 자동으로 실행하고 결과를 표시할 수 있습니다. 이를 통해 실시간으로 코드 품질을 유지할 수 있습니다.

##### ESLint 사용 방법:

1. 설치: 프로젝트의 루트 디렉토리에서 ESLint를 설치합니다.
2. 설정: .eslintrc 파일을 프로젝트 루트에 생성하여 ESLint 설정을 구성합니다. 이 파일에서 사용할 규칙, 플러그인 및 확장 프로그램을 설정할 수 있습니다.
3. 규칙 설정: 프로젝트에 적합한 ESLint 규칙을 설정합니다. 이는 코드 스타일, 오류 수준 등을 조절하여 프로젝트 요구 사항에 맞게 사용할 수 있습니다.
4. 통합: 대부분의 개발 도구에서 ESLint를 통합하여 코드 편집 중에 실시간으로 경고 및 오류를 표시할 수 있습니다. 예를 들어, Visual Studio Code에서는 ESLint 확장 프로그램을 설치하여 이 기능을 활용할 수 있습니다.
5. 실행: 명령줄에서 eslint 명령을 사용하여 전체 프로젝트나 특정 파일에 대한 ESLint 검사를 실행할 수 있습니다. 이를 자동화하여 빌드 스크립트에 통합할 수도 있습니다.
6. 결과 해석: ESLint는 코드에서 발견된 문제를 목록으로 출력합니다. 이를 통해 문제를 해결하고 코드 품질을 향상시킬 수 있습니다.
ESLint는 JavaScript 및 TypeScript 프로젝트에서 품질을 향상시키고 일관된 코드를 유지하는 데 중요한 역할을 합니다. 이를 통해 코드의 가독성을 높이고 유지 보수성을 향상시킬 수 있습니다.

### 3-3. 주요기능 : VSCode의 통합 터미널 (고민혁)

##### 통합 터미널이란?

- VSCode 내부에 내장된 커맨드라인 인터페이스
- 별도의 커맨드 라인 툴을 사용하지 않고도, VSCode 내 오픈된 프로젝트에 관한 코드 작성, 빌드 테스트 및 디버깅 등을 지원


##### 통합 터미널의 장점

- Windows, macOS, Linux 등 다양한 운영 체제에서 사용할 수 있어, 개발자들 사이의 작업 호환성을 높일 수 있다.
- 통합 터미널 내에 VSCode에 내장된 Git 명령어들을 입력하여 프로젝트의 버전 관리 작업을 수월하게 할 수 있다.
- 다양한 확장 프로그램(Extension)을 지원하여 기능 확장에 유연하며, 확장 프로그램을 설치하기만 하면 특정 프로그래밍 언어의 고유한 명령어를 사용할 수 있습니다.


##### 실무 사용 예시 

 #### 프론트엔드 개발자

 - 통합 터미널에서 npm 혹은 yarn과 같은 패키지 매니저를 사용하여 프로젝트의 의존성을 관리할 수 있다.
 - 통합 터미널을 통해 자바스크립트 기반의 라이브러리 혹은 프레임워크에서 임시 서버를 실행시켜 실시간으로 UI 및 비지니스 로직을 테스트할 수 있다.
 - 통합 터미널을 통해 프로젝트 빌드 명령어를 실행시켜 배포 가능한 웹 애플리케이션 형태를 확인 할 수 있다.


 #### 백엔드 개발자

 - 통합 터미널을 통해 로컬 서버를 실행하고 관리 할 수 있다. 이를 통해 API 개발을 진행하면서 실시간으로 변경 사항을 체크하고 로그를 모니터링할 수 있다.
 - 통합 터미널을 통해  MySQL 혹은 MongoDB와 같은 데이터베이스와 연동된 프로젝트의 데이터 관리를 진행할 수 있다

##### VSCode와 Git의 통합이란?

- GIt과의 통합은 개발자들이 코드 작성 환경을 벗어나지 않고도 Git을 사용하여 작업할 수 있도록 해준다.
- VSCode에서 작업합 프로젝트의 버전 관리를 용이하게 한다.
- 개발자는 코드의 변경 이력을 관리하고 다른 개발자와의 협업을 수월하게 할 수 있다.

##### VSCode의 내장된 Git을 활용한 세부 기능

###### 소스 제어 패널

- 코드의 버전 관리와 관련된 작업을 수행할 수 있는 패널이다.
- VSCode에는 소스 제어 패널이 내장되어 있어서 Git 저장소의 상태를 확인할 수 있다.

###### - 소스 제어 패널의 장점

  - 코드 변경 이력을 체계적으로 관리하고 작업 흐름을 효율적으로 관리할 수 있다.
  - Git 작업에 필요한 다양한 기능에 대한 바로가기를 제공해 작업을 빠르게 수행할 수 있다.

###### 원격 저장소 관리(푸시(push) 및 풀(pull))

- 로컬 저장소와 연결된 원격 저장소로 변경 내용을 푸시하거나, 원격 저장소에서 변경 내용을 풀하여 로컬 저장소로 가져올 수 있다.
- 푸시나 풀 작업은 간단한 명령을 통해 VSCode 내에서 수행할 수 있다.

###### - 푸시(push)의 장점
-
  - 푸시를 통해 로컬 저장소의 변경사항을 원격 저장소에 업로드할 수 있다. 이를 통해 다른 개발자들과 코드를 공유하고 협업 가능하다.
  - 개발자들이 로컬에서 작업한 변경사항을 원격 저장소로 푸시함으로써 모든 팀원이 최신 코드를 유지해 동기화를 유지할 수 있다.
  - 원격 저장소에 코드를 푸시하여 중요한 코드 변경사항을 안전하게 백업할 수 있다.

###### - 풀(pull)의 장점
-
  - 풀을 통해 원격 저장소의 변경사항을 로컬 저장소로 가져올 수 있다. 이를 통해 다른 팀원들이 푸시한 변경사항을 로컬에 적용하여 최신 코드를 유지할 수 있다.
  - 다른 개발자들이 작업한 변경사항을 풀하여 로컬에서 확인하고 검토할 수 있습니다. 이를 통해 코드 리뷰를 수행하고 품질을 향상시킬 수 있다.
  - 풀을 통해 변경사항을 가져오기 전에 충돌이 있는지 미리 확인할 수 있다.

##### VSCode와 Git의 통합이 가지는 의의
- 이는 코드 편집과 버전 관리를 하나의 환경으로 통합하여 개발자들이 작업 흐름을 효율적으로 관리하고 협업할 수 있도록 지원한다. 이를 통해 개발자들은 실시간으로 변경 사항을 공유하고 리뷰하며, 코드의 변경 사항을 체계적으로 추적하고 관리하여 프로젝트의 일관성과 안정성을 유지할 수 있다. 이러한 점들은 개발 프로세스의 효율성을 향상시키고 소프트웨어 프로젝트의 성공을 이끌어냅니다. 이에 VSCode와 Git의 통합은 현대 소프트웨어 개발에서 핵심적인 역할과 가치를 지닌다고 볼 수 있다.

### 3-5. 주요기능 : Visual Studio Code의 자동완성 기능 (황어진)

##### 스니펫이란?

- VSCode에서 스니펫(Snippets)은 재사용 가능한 소스 코드, 기계어, 텍스트의 작은 부분을 일컫는 프로그래밍 용어이다. 자주 사용하는 소스 코드를 미리 정의하여 지정해 둔 예약어로 한 번에 불러올 수 있는 기능을 말한다.  즉, 소스 코드 자동 완성이다.

- 프로그래머들은 일반적으로 자주 사용하는 코드 패턴이나 알고리즘을 빠르게 삽입하기 위해 코드 스니펫을 사용한다.

 ##### 장점

- 이러한 기능은 개발자가 계속 반복해서 입력해야 하는 작업에서 시간을 단축시켜주는 역할을 한다. 개발 과정을 가속화하고, 오류를 줄이는 데도 도움이 된다.

  ##### 코드 스니펫의 예

 - 코드 스니펫은 다양한 프로그래밍 환경에서 사용될 수 있으며, 특정 작업을 수행하는 데 필요한 최소한의 코드 양을 포함한다.
HTML 태그, 데이터 베이스에 연결하는 스크립트, 특정 알고리즘을 구현하는 코드 등이 코드 스니펫의 예가 될 수 있다.

  ##### 구분

- 코드 조각은 VS Code에서 자체적으로 제공하는 내장 코드 조각과, 사용자가 직접 정의한 사용자 정의 코드 조각으로 구분할 수 있다.

  ##### 내장 코드 조각 (Built-in Snippets)

  - VS Code는 자체적으로 다양한 언어에 대한 내장 코드 조각을 제공한다. HTML에서 느낌표를 입력하고 탭을 누르면 기본적인 HTML 파일 구조가 만들어진다. 이렇게 내장된 코드 조각을 사용하려면 해당 언어에서 스니핏 예약어를 입력하고 Tab 키를 누르면 VS Code가 자동으로 코드 조각을 완성해준다.

  ##### 사용자 정의 코드 조각 (User Snippets)

  - 사용자 정의 코드 조각은 기본적으로 제공하는 코드 조각 이외에 개발자가 자주 사용하는 코드 패턴이나 템플릿을 미리 정의하여 등록한 것을 말한다. 사용법은 내장 코드 조각과 같다.

  body : 실제 화면에 출력되는 부분이다. [] 사이에 작성하며, 따옴표로 묶어서 적어주고, 줄바꿈 시 콤마(,)를 삽입해야한다. 들여쓰기를 하려면 공백을 입력해야 한다. HTML 코드의 따옴표는 (/")로 표기한다. $1, $2는 탭 정지를 나타낸다. $0은 스니펫이 끝난 후 최종 커서 위치를 나타낸다.
